# Encounter ID — RHIE API Analysis

Documentation of external API usage in the Encounter ID module.

---

## Summary

**The PHP Encounter ID module makes zero external API calls.**

Encounter IDs are generated locally as UUIDs and stored in Medisoft facility databases. No HTTP requests are sent to the Rwanda HIE during `generate_encounters_batch.php` execution.

---

## Platform Config Note

Phase 1 platform config includes:

```yaml
rhie:
  encounterIdPath: /encounters/id
```

The `@rhie/rhie-client` package exposes `generateEncounterId()` for potential future use, but **this endpoint is not invoked by the PHP reference implementation**.

| Component | Uses encounterIdPath? |
|-----------|----------------------|
| `generate_encounters_batch.php` | No |
| `EncounterController.php` | No |
| `EncounterModel.php` | No |

---

## Downstream HIE Usage (Out of Scope)

Encounter IDs generated by this module are **consumed** by separate upload services:

| Service | PHP Batch | HIE Endpoint | Uses encount_id |
|---------|-----------|--------------|-----------------|
| Visit Encounter | `upload_visit_encounters_batch.php` | POST Encounter (visit) | Yes |
| Observations | `upload_encounters_batch.php` | POST Observation | Yes |
| Transfer | Upload controllers | POST Encounter (transfer) | Yes |

These are Phase 5+ services, not Encounter ID generation.

## TypeScript Encounter ID Service

### Production mode actions

1. Query facility database
2. Build encounter insert records (local payloads)
3. INSERT into `encounter_main` / `encounter_patients`
4. UPDATE source table `rhie_status`

### Shadow mode actions

1. Query facility database
2. Build encounter insert records
3. Log payloads
4. **Skip** INSERT and UPDATE

### RHIE client dependency

**Not required** for Encounter ID service. The service depends on:

- `@rhie/database` — SQL execution
- `@rhie/shared` — UPID utilities
- `@rhie/config` — service configuration
- `@rhie/worker-framework` — worker lifecycle

Do not import `@rhie/rhie-client` in Encounter ID processor unless a future ADR adds HIE pre-registration.

---

## Authentication

N/A for this module. No credentials, headers, or retry logic apply.

---

## Error Handling

No HTTP status codes. Errors are:

- SQL exceptions (connection, constraint violations)
- Empty result sets (normal — no work to do)

PHP catches exceptions per generator and continues. TypeScript mirrors this pattern.

---

## Validation Strategy

Since there is no HIE API:

| Validate | Method |
|----------|--------|
| Selection SQL | Exact string match with PHP |
| Insert payloads | Unit tests on `EncounterPayloadBuilder` |
| Status updates | Repository method tests with mocked DB |
| End-to-end | Shadow mode logs compared against PHP fixture rows |

Payload comparison uses deterministic inputs with mocked UUID generator in tests.
